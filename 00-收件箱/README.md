# 00-收件箱

此目录是 **唯一入口**——任何新增的资料、笔记、文件、链接、想法，**全部先丢到这里**。你不需要思考分类，后续由 LLM 处理并归档。

## 使用流程

```
捕获 → 放入 00-收件箱/ → 告诉 LLM 处理 → LLM 分类并归档 → 进入知识库
```

## 可以放什么

| 类型 | 示例 |
|------|------|
| 网页文章 | Obsidian Web Clipper 剪藏的 markdown |
| 学术论文 | PDF、markdown 摘要 |
| 图片/附件 | 截图、PDF、数据文件 |
| 灵感速记 | 临时想法、语音转文字 |
| 会议/播客笔记 | 原始记录 |
| 任意链接 | 收藏的网址、待读文章 |

## 你对 LLM 说的话

只要涉及理解、处理、入库，都可以说：

- "读取并整理这个文件"
- "分析这篇文章"
- "提炼核心观点"
- "总结并归档"
- "处理收件箱"

## LLM 的处理逻辑

1. **读取** `00-收件箱/` 中的文件
2. **判断类型**：
   - 网页文章/博客 → 移动到 `10-原始资料/01-文章/`
   - 学术论文 → 移动到 `10-原始资料/02-论文/`
   - 图片/PDF/数据 → 移动到 `10-原始资料/03-资源/`
   - 你自己的笔记/灵感 → 直接在知识库中创建或补充页面
3. **重命名**为规范格式：`YYYY-MM-DD-slug.md`
4. **执行摄取**：创建来源摘要、更新实体/概念/主题/索引/日志/统计
5. **清理收件箱**：处理完的文件从 `00-收件箱/` 移除
