# 人保张鹏飞面谈沟通_原文

> 继首次沟通后的第二次面谈，张鹏飞深入剖析了农险承保端数据真实性困境、遥感技术精度局限性以及"保障救赔"全链条服务模式构想。

> **原始资料**：[[10-原始资料/07-会议沟通/20240719-人保张鹏飞面谈沟通_原文.md|全文]]

## 核心观点

1. **承保数据失真是最大痛点**：村委提供的种植清单往往与粮食补贴数据相关联，存在虚报瞒报。保险公司抽样验标无法改变承保总量结果，本质上是"走流程"。一个县水稻承保数据从1100万亩被审计退至970万亩，真实情况远比官方数据严峻。
2. **耕地面积持续下降**：广东耕地从3150万亩降至2820万亩，降幅250万亩，但保险承保数据比实际耕地面积还大。大量水田实际已抛荒（阳西县官方粮食数据十几万至二十万亩，仅保了八千多亩）。
3. **遥感精度争议**：招标文件要求3米卫星影像90%精度提取水稻，张鹏飞认为是"技术伪命题"——广东云层厚导致春夏无可用影像，光谱提取易将草地/树木误判为水稻，80%精度已是极限。
4. **全链条服务模式**：从风险区划地图 → 精准承保（地块底层数据） → 监测预警（台风、病虫害） → 防灾减损（农机调度、抢收） → 快速理赔，构建完整的"保障救赔"闭合链条。
5. **差异化定价需数据支撑**：风险费率差异化目前只做到森林险层面（费率差一倍），主粮作物仍统一费率。国家减灾中心的历史灾害数据可为差异化定价提供量化依据。

## 关键讨论要点

- 广东水稻业务人力成本超50%（因地块细碎，需大量人工），投入产出比极低
- 人保广东5年积累5000万亩实地验标数据，年验标3-5万个点（远超国家统计局广东调查队的2000个抽样点）
- 承保操作流程：村委收集清单 → 保险公司按清单验标（抽样3-5户/村） → 勾图确认 → 校验村总面积
- 土地流转率高达80%/年，地块权属频繁变更导致数据难以维护
- 建议推广"农机农服"模式：通过农机调度掌握真实种植数据

## 结论与后续

- 核心卡点在于缺乏权威、统一、可用的农业基础数据（种植一张图）
- 张鹏飞建议以水田底图作为基础风控线：实际水稻种植不能超过水田面积
- 平台设计需围绕"承保数据源→监测预警→农业生产服务→快速理赔"的主线
