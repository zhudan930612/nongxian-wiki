# 20260523-农险业务沟通-林老师

> 邀请林老师对农保云四层架构进行技术可行性评审，同时深入讨论产品定位、商业模式、数据战略和竞争壁垒，明确从"信息化公司"向"保险科技公司"升级的战略方向。

## 核心观点

1. **架构逻辑成立**：在现有信息化平台上叠加AI中台，不动底层，保护客户投资，技术可行。
2. **交付的是应用层（智能体）**：中台是自己用的工具，客户买的是解决具体问题的智能体应用。
3. **数据是核心壁垒**：静态数据（地块、保单）+ 动态数据（遥感、气象、信息员采集）+ 知识库（规则、模型），三层数据构建护城河。
4. **120万灾害信息员是独特资产**：这是竞品无法复制的地面网络，是业务闭环的关键。
5. **定位保险科技（FinTech）而非农业信息化**：垂直赛道+保险认知+AI能力，估值和发展空间完全不同。

## 关键内容

### 架构评审（林老师视角）
- 四层架构逻辑正确：业务数据层 → AI中台 → 智能体应用 → 线下协同。
- 中台不是交付物，是内部工具；交付给客户的是应用层的智能体。
- 参考Coze（扣子）模式：中台提供编排能力，上层生成具体应用/智能体。
- 数据层应拆分为：数据源层（原始静态数据）→ 知识库层（清洗关联后的数据）→ 应用层。
- 动态数据包括：灾害信息员采集、无人机、遥感、传感器等实时数据，这是最大价值补充。

### 产品定位与商业模式
- 不做管理系统，做业务系统：直接嵌入保险公司业务流程，而非仅供领导查看的数据大屏。
- 三种切入方式：
  - 从下切：补充数据（如农业局数据）。
  - 从中切：叠加AI中台和智能体。
  - 从上切：直接灌入完整解决方案（如小麦过冬专项）。
- 收费模式：基础开发费 + 按效果分成（降赔分成），降低客户决策门槛。

### 竞争壁垒分析
- 佳格天地：从农业政府端切入，保险业务认知弱。
- 珈和科技：武汉大学遥感背景，技术强但缺乏保险场景理解。
- 平安鹰眼：服务好但局限于平安体系，其他保险公司不会用。
- 华浙智擎优势：保险行业人脉 + 国家减灾中心数据/无人机/信息员网络 + 浙大技术合作。

### 数据战略
- 补齐保险公司没有的动态数据：气象、遥感、灾害信息员实时采集。
- 数据可延展至农商行农户授信、公安反欺诈等场景。
- 知识库构建是难点：如何把专家经验（如道德风险识别规则）转化为可计算的知识库。

### 行业背景与时机
- 农险增速放缓（2024年约7.6%），财政补贴收紧，保险公司从抢增量转向保存量、提质量。
- 精细化经营时代到来，降本增效需求迫切，正是科技切入的好时机。
- 农险市场集中度高：人保约40%，太保、中华联合各约10%几，国寿财、平安约8%。

### 落地节奏
- 以浙江太保为试点，目标6月完成道德风险智能体，9月在太保全国风险减量会议上发布。
- 同步拓展应急场景（城市内涝预警），验证AI中台的跨领域复用能力。
