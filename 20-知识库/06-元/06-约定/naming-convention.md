# 命名约定

> 本页汇总知识库的所有命名规则和文件操作守则。

## 目录命名

- 一级和二级目录使用**中文**
- 示例：`00-收件箱/`、`10-原始资料/01-政策法规/`、`20-知识库/00-索引/`
- `XX-` 前缀用于排序和分类，**不用于页面标题**

## 文件名

### 原始资料文件

- 格式：`YYYYMMDD-原标题.扩展名`
- 保留原始资料的标题，中英文均可
- 示例：`20260625-案例|遥感在农险应用中的全流程解析.md`

**日期优先级**：
1. 文档有自身固有日期（`published` 字段、发文日、会议日期等）→ 使用该日期
2. 无固有日期 → 使用 frontmatter 中 `created` 日期

### 知识库页面文件

#### 实体、概念、主题页面

- 位于 `01-实体/`、`02-概念/`、`03-主题/` 下
- **文件名必须带编号前缀**：`01-`、`02-`、`03-`，与目录前缀一致
- 前缀后接页面主题，中文、英文或缩写均可，简洁明确
- 示例：
  - `01-实体/01-太保产险.md`
  - `02-概念/02-双精准.md`
  - `02-概念/02-NDVI.md`
  - `03-主题/03-遥感在农业保险中的应用.md`

**业务子目录**：允许在实体/概念/主题目录下按业务需要建立子目录（如 `01-实体/客户现状/`），子目录内的文件仍需带 `01-`/`02-`/`03-` 前缀。

#### 来源摘要页

- 位于 `04-来源/` 下，按原始资料分类镜像子目录
- 格式：`04-来源/XX-分类/YYYYMMDD-原标题.md`
- 部分分类含三级子目录，如 `04-来源/06-媒体报道/01-实务经验/YYYYMMDD-标题.md`
- 文件名可与对应原始资料相同，便于对应

#### 问答页

- 位于 `05-问答/` 下
- 格式：`YYYYMMDD-问题简述.md`
- 示例：`20260626-保险公司的核心痛点.md`

### PDF 归档目录

PDF 原始文件由 ingest skill 统一归档到按篇隔离的目录：

```
10-原始资料/99-PDF原件/{分类}/YYYYMMDD-标题/
├── YYYYMMDD-标题.pdf
└── images/
```

## 页面标题（h1）

- 每个页面第一个 `#` 标题使用**中文**
- 示例：`# LLM Wiki 方法论`，文件名为 `llm-wiki.md`

## 编号前缀系统

| 前缀 | 分类 | 内容 |
|------|------|------|
| `00-` | 系统 | 索引、日志、统计 |
| `01-` | 实体 | 人物、组织等实体页面 |
| `02-` | 概念 | 定义、理论、方法论 |
| `03-` | 主题 | 跨实体的综合分析、专题 |
| `04-` | 来源 | 原始资料摘要 |
| `05-` | 问答 | 归档的查询结果 |
| `06-` | 元 | 模板、待办、约定 |

## 文件操作守则

### 路径格式

- 始终使用**相对路径**（相对于 vault 根 `E:/wiki/nongbaoyun/`）
- 引用 vault 内文件时使用：`路径/文件名.md`
- **不要**使用绝对路径或前导斜杠

### 修改规则

- `10-原始资料/` 下的文件**永不修改**（只读）
- `20-知识库/` 下的文件由 LLM 全权维护
- `00-收件箱/` 下的文件处理后可删除或移入原始资料

### 引用风格

- 在回答中使用 Wiki 链接指向知识库页面：`[[02-概念/02-双精准|双精准]]`
- 对原始资料使用 YYYY-MM-DD 格式日期关联
