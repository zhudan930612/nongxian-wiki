# 知识图谱增强设计

> 在现有 LLM Wiki 知识库基础上，增加类型化关系层，将平面 `[[链接]]` 升级为带语义的知识图谱，提升查询质量和实体关联发现能力。

## 背景

当前 vault 已有 25 个实体和 29 个概念页面，页面间通过 `[[链接]]` 关联。但现有链接只是"相关"，没有语义类型——看不出一条连接是"研究关系"、"合作 关系"还是"隶属关系"。这导致：
- 查询时无法做定向图遍历（只能全文搜索）
- 跨实体关系需要人工阅读才能发现
- 无法统计"谁和谁合作最多"、"谁的研究领域最广"等结构化问题

## 关系类型定义

### 核心关系（10 种）

| 关系 | 反向关系 | 适用场景 | 示例 |
|------|---------|---------|------|
| `研究` | `被研究` | 学者 → 概念/主题 | 易福金 → 农业保险 |
| `任职于` | `有成员` | 人物 → 组织 | 易福金 → 浙江大学 |
| `合作` | `合作` | 组织/人物 ↔ 组织/人物（双向） | 浙江大学 → 太保产险 |
| `监管` | `被监管` | 监管机构 → 被监管方 | 国家金融监督管理总局 → 太保产险 |
| `竞争` | `竞争` | 公司 ↔ 公司（双向） | 珈和科技 → 佳格天地 |
| `提供产品` | `使用产品` | 公司 → 产品/平台 | 珈和科技 → 农险通 |
| `制定政策` | `受政策影响` | 政府/监管 → 政策/制度 | 国家金融监督管理总局 → 完全成本保险 |
| `隶属` | `有下属` | 组织 → 子组织 | 广东人保财险 → 中国人民财产保险股份有限公司 |
| `案例/试点` | `试点参与` | 组织 → 地点/项目 | 太保产险 → 孝感市孝南区 |
| `关注/痛点` | `是痛点` | 组织 → 概念（表示该组织面临此问题）| 广东人保财险 → 道德风险 |

### 关系记录格式

每条关系包含以下字段：

```yaml
- source: 01-实体/01-易福金        # 源实体（必须使用标准路径格式，不含 20-知识库/ 前缀）
  type: 研究                         # 关系类型（中文）
  target: 02-概念/02-农业保险        # 目标实体
  sources:                           # 证据来源
    - 04-来源/02-学术论文/20260625-《如何看待和解决农业保险双精准问题》（庹国柱、易福金）（P15-21）
  confidence: 0.85                   # 该关系的置信度（继承自来源页置信度，或 lint 计算）
  bidirectional: false               # 是否为双向关系
```

## 工作流改造

### Ingest 时抽取关系

在现有 Ingest 工作流中增加关系抽取步骤（放在步骤 9a"增量字段更新"之后）：

```
### 9b. 知识图谱关系抽取

创建来源摘要页时，扫描正文中的实体关系模式，自动生成关系条目：

1. **识别关系动词**：在正文中搜索以下模式：
   - "[实体A]研究[概念B]" → type: 研究
   - "[实体A]任职于[实体B]" → type: 任职于
   - "[实体A]与[实体B]合作" → type: 合作
   - "[实体A]监管[实体B]" → type: 监管 / 制定政策
   - 显式关系声明（如"据XX介绍"、"XX公司是YY的下属机构"）
2. **去重**：检查 knowledge-graph.md 中是否已有相同的关系记录
3. **追加**：将新关系追加到 knowledge-graph.md 的关系列表末尾
4. **置信度继承**：新关系的 confidence 取来源页 confidence 值的 80%（待 lint 校正）
```

### Query 时图遍历

在 Query 工作流中的"搜索"步骤扩展如下：

```
原搜索流程：
  搜索 index.md + 全文搜索相关页面 → 阅读 → 综合回答

扩展后的搜索流程：
  1. 搜索 index.md + 全文搜索相关页面
  2. **知识图谱辅助搜索**：
     a. 确定问题中涉及的实体
     b. 在 knowledge-graph.md 中找到这些实体的关系
     c. 沿相关关系类型行走（如"合作"、"研究"）
     d. 发现通过纯文本搜索无法直接找到的相关页面
  3. 综合搜索结果和图谱结果 → 阅读 → 综合回答
```

例如问"浙大在农险领域有什么布局"，流程如下：
1. 搜索"浙江大学"等关键词
2. 在知识图谱中找到浙大 → 沿"有成员"（易福金）→ 沿"研究"（农业保险）、沿"合作"（太保产险）
3. 结合两路结果回答，比纯搜索多覆盖合作方和关联研究等间接信息

### Lint 时图谱验证

在 Lint 工作流中增加步骤：

```
### 图谱完整性验证

1. **孤立实体检测**：在 knowledge-graph.md 中查找未出现在任何关系中的实体/概念
   - 列出孤立实体页面
   - 建议：基于页面内容推断可能的关系类型和目标
2. **残缺关系检测**：检查关系是否有 source、type、target、sources 四个必要字段
   - 报告缺少证据来源的关系
3. **单向关系合理性检查**：对于 bidirectional: false 的关系，判断是否需要双向
   - 如"研究"关系通常只需要单向（A 研究 B），"合作"需要双向
   - 报告不合理的单向设置
4. **置信度衰减**：检查关系的 last_confirmed 字段，超过 180 天的关系 confidence 自动乘以 0.8
5. **图谱统计数据更新**：在 stats.md 中新增图谱统计字段
```
