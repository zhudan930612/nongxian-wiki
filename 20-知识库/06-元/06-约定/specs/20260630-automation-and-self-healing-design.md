# 自动化与自愈增强设计

> 在现有 LLM Wiki 知识库基础上，增加自动化钩子和质量自愈能力，将手动操作降到最低，让 lint 从"检查报告"升级为"自动维护"。

## 背景

当前 vault 的三个核心工作流（Ingest/Query/Lint）都需要用户手动触发。lint 目前能发现问题，但不会自动修复。本设计将两者合并推进，因为它们共享相同的机制——hooks 和自愈逻辑。

---

## A. 自动化钩子

### A1. 自动上下文加载（Session Start）

当 Claude Code 在 vault 中启动时，自动执行以下操作：

1. 读取最近 5 条 log 条目 → 了解 vault 最近动态
2. 检查 `00-收件箱/` 是否有待处理文件 → 如果有，主动提醒用户
3. 读取 `stats.md` → 了解 vault 整体健康状况
4. 检查是否有 `status: fading` 的页面 → 提醒是否需要更新

**实现方式**：通过 `.claude/settings.json` 的 `onSessionStart` hook 执行一个轻量级检查脚本。

### A2. 定时 lint 提醒

利用 Claude Code 的 cron 能力，每周自动执行一次 lint 检查，无需用户记住。

- 频率：每周一次（建议周日晚间或周一早晨）
- 触发时：自动执行 `llm-wiki-lint`，并通知用户检查结果
- 如果 lint 发现问题：主动报告摘要，等待用户指示
- 如果 lint 无问题：静默完成，仅更新 stats 中的"最后检查"日期

**实现方式**：通过 CronCreate 创建每周定时任务。

### A3. 会话结束结晶增强

目前 Query 工作流已有结晶机制，但只覆盖"用户提问 → 回答"场景。扩展覆盖：

1. **长讨论自动结晶**：当一次对话包含 5+ 轮以上的讨论且涉及知识库主题时，自动触发结晶
2. **代码/方案讨论结晶**：当讨论产生可复用的结论或设计决策时，自动归档
3. **会议记录后处理**：摄入会议记录后，自动抽取关键决策和待办事项

**实现方式**：在 CLAUDE.md 中扩展结晶触发条件（已在阶段 1 中部分实现），并在对话中主动识别结晶时机。

### A4. 收件箱主动提醒

当 `00-收件箱/` 中有新文件时，在当前会话中主动提醒用户，而不是等用户说"处理"。

- 会话开始时检查收件箱状态
- 如果有文件且用户有空，主动询问是否要处理
- 如果用户说"稍后"，记录到 todo.md

## B. 质量与自愈

### B1. 孤立页面自动修复

当前 lint 会报告孤立页面（无入站链接的页面）。升级为自动修复：

1. 读取孤立页面的内容，理解其主题
2. 在全 vault 中搜索相关页面（关键词匹配、实体匹配）
3. 在合适的相关页面中添加 `[[孤立页面]]` 链接
4. 如果完全找不到合适位置，在孤立页面中添加 `## 相关页面` 区段，手动标记待处理

### B2. 矛盾自动裁决

当 lint 发现两个页面存在矛盾时，不再仅报告，而是自动给出裁决建议。裁决依据（按优先级）：

| 优先级 | 依据 | 说明 |
|--------|------|------|
| 1 | **来源权威性** | 政策文件 > 学术论文 > 行业报告 > 媒体报道 > 会议沟通 |
| 2 | **来源时效性** | 更新日期越近越可信 |
| 3 | **置信度** | 页面 confidence 越高越可信 |
| 4 | **支持来源数** | 支持某一方的来源数量更多 |

裁决结果格式：
```
矛盾：页面A 说"X"，页面B 说"Y"
裁决：采用页面B 的说法
理由：页面B 引用了2025年新政（权威性：政策文件），页面A 仅引用2022年旧版
操作：已更新页面A 为 status: superseded，添加 superseded_by 指向页面B
```

### B3. 低置信度页面自动改进

对 `confidence < 0.4` 的页面自动触发改进流程：

1. 检查原因：是来源不足？时效性过期？内容不完整？
2. 针对性地改进：
   - 来源不足 → 在全 vault 中搜索可补充的来源链接
   - 时效性过期 → 标记为 `status: fading`，降低查询优先级
   - 内容不完整 → 补充基本结构（h1、摘要、相关链接）
3. 无法自动改进的 → 在 lint 报告中标记为"需人工干预"

### B4. 链接健康自动修复

1. **断裂链接修复**：`[[链接]]` 指向不存在的页面时：
   - 如果链接文本和某个已有页面匹配，自动修正路径
   - 如果频繁引用（≥3次），自动创建页面
   - 如果引用次数少，记录到 lint 报告

2. **冗余链接清理**：同一页面内多次链接同一目标 → 保留第一次，后续清理

3. **别名一致性检查**：
   - 例如 `[[01-人保]]` 和 `[[01-中国人民财产保险股份有限公司]]` 指向同一实体
   - 自动统一为标准名称，或添加 aliases

### B5. 自愈工作流集成到 Lint

上述所有自愈操作集成到 Lint 工作流中，形成一个"检查 → 修复 → 报告"的闭环：

```
当前 Lint：
  扫描 → 发现问题 → 报告问题 → 等待用户处理

自愈 Lint：
  扫描 → 发现问题
    ├─ 能自动修复 → 执行修复 → 在报告中说明"已修复"
    └─ 不能自动修复 → 报告问题 → 标注"需人工处理"
```

## 实施优先级建议

| 优先级 | 项目 | 难度 | 价值 | 说明 |
|--------|------|------|------|------|
| P0 | **自动上下文加载** | 低 | 高 | 每次启动节省 10 秒 + 不会遗漏收件箱 |
| P0 | **定时 lint 提醒** | 低 | 高 | vault 保持健康，不需要你记 |
| P1 | **孤立页面自动修复** | 中 | 中 | 减少灰色链接、提升联通性 |
| P1 | **断裂链接修复** | 中 | 中 | 保持链接完整性 |
| P2 | **矛盾自动裁决** | 高 | 中 | 需要复杂判断逻辑 |
| P2 | **低置信度页面改进** | 高 | 低 | 当前 vault 所有页面 confidence 均为 0.75，尚无低分页面 |
| P3 | **会话结束结晶增强** | 低 | 低 | 当前结晶机制已覆盖主要场景 |

---

## CLAUDE.md 修改要点

1. **新增"自动化"章节**：描述自动上下文加载、定时 lint、收件箱提醒
2. **Lint 工作流升级**：将"检查"改为"检查与自愈"，增加自动修复步骤
3. **Query 工作流扩展**：增加讨论自动结晶

## 实施范围总结

| 模块 | 涉及文件 | 改动量 |
|------|---------|--------|
| settings.json hook | `.claude/settings.json`（新建或修改） | 小 |
| 定时 lint | CronCreate 任务 | 小 |
| CLAUDE.md | 新增自动化章节 + Lint 自愈升级 | 中 |
| lint skill | 增加自愈步骤（孤立页面修复、矛盾裁决、断裂链接修复） | 中 |
