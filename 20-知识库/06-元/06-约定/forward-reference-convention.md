# 前向引用约定

> **前向引用（Forward Reference）**：写 Wiki 链接时指向一个尚未创建的页面。Obsidian 将其渲染为**灰色链接**，表示"这个页面被引用了，但尚未创建"。

## 通用链接语法

- **内部链接**：使用 Wiki 链接格式 `[[页面名]]` 或 `[[路径/页面]]`
- **外部链接**：使用 Markdown 格式 `[显示文本](url)`
- **图片嵌入**：使用 `![[图片文件名]]`

## 为什么用前向引用

没有前向引用时，LLM 在 Ingest 阶段面临两难：

- 跳过链接 → 丢失知识关联
- 立即创建目标页 → Ingest 膨胀为 10-20 次页面操作

前向引用将这两个阶段分离：

```
Ingest 阶段：只管写链接，不关心目标是否存在
Lint 阶段：  统计灰色链接，批量创建高频引用页面
```

## 链接格式规范

所有 Wiki 链接必须使用以下格式：

| 目标类型 | 正确格式 | 错误格式 |
|---------|---------|---------|
| 实体 | `[[01-实体/01-实体名称]]` | `[[01-实体/实体名称]]`、`[[实体名称]]` |
| 概念 | `[[02-概念/02-概念名称]]` | `[[02-概念/概念名称]]`、`[[概念名称]]` |
| 主题 | `[[03-主题/03-主题名称]]` | `[[03-主题/主题名称]]` |
| 来源 | `[[04-来源/XX-分类/YYYY-MM-DD-slug]]` | `[[04-来源/YYYY-MM-DD-slug]]` |

**规则要点：**
1. **不使用完整路径**（即不以 `20-知识库/` 开头）
2. **必须带前缀编号**（`01-`、`02-`、`03-`、`04-`）
3. **禁止无前缀裸名**：`[[双精准]]` ❌ → `[[02-概念/02-双精准]]` ✅
4. **来源链接必须包含分类子目录**：
   - 二级子目录示例：`[[04-来源/02-学术论文/20260625-基于多源数据的全国三大主粮生产风险评估研究]]`
   - 三级子目录示例：`[[04-来源/06-媒体报道/01-实务经验/20260626-农险地块信息库试点：方向对了，落地才是关键]]`

## Ingest 阶段规则

创建来源摘要页或其他新页面时：

1. **引用实体、概念、主题时，直接写 `[[链接]]`，不检查目标页面是否存在**
2. 如果目标已存在 → 链接自动变蓝（正常）
3. 如果目标不存在 → 灰色链接，属于正常状态
4. **不**在 Ingest 时批量创建目标页面
5. **例外**：如果缺失页面是本次 Ingest 的核心主题，且 LLM 有足够信息创建，可以同时创建

**简单原则**：写链接时像所有页面都存在一样去写。

## Lint 阶段规则

每次执行 `llm-wiki-lint` 时，增加以下步骤：

### 1. 扫描灰色链接

扫描整个 `20-知识库/` 下所有 Wiki 链接，找出指向不存在页面的链接。

### 2. 按引用次数分类

| 引用次数 | 处理方式 |
|---------|---------|
| **≥ 3 次** | 自动创建页面，补充 YAML frontmatter 和基本结构 |
| **1-2 次** | 记录在 lint 报告中，标记为"低优先级" |
| **用户指定高优** | 即使引用 1 次也创建 |

### 3. 批量创建页面

对引用 ≥ 3 次的缺失页面，自动创建并补充：
- YAML frontmatter（title, created, updated, tags）
- 基本页面结构（`# 标题`、`> 一句话描述`、`## 相关来源`）
- 反向链接到引用它的来源页

### 4. 更新统计

更新 `stats.md` 中的"待创建实体数"和"待创建概念数"。

## 统计跟踪

`stats.md` 新增字段：

```
| 待创建实体（灰色链接） | N |
| 待创建概念（灰色链接） | M |
```

每次 Lint 后更新。趋势观察：如果数字持续下降，说明前向引用在正常消化；如果持续上升，说明 Ingest 速度超过了 Lint 清理速度，需要增加 Lint 频率。

## 工作流映射

```
┌─────────────────────────────────────────────┐
│                  Ingest                       │
│  创建来源摘要 → 写 [[链接]]（不管是否存在）       │
│  结果：来源页有灰色链接 ✅（正常状态）             │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│                  Lint（定期）                  │
│  扫描全部灰色链接 → 统计引用次数               │
│  ≥3次 → 批量创建  1-2次 → 记录备查           │
│  更新 stats.md 中的待创建数                   │
└─────────────────────────────────────────────┘
```

## 常见问题

**Q：Ingest 时创建了来源摘要，但里面的链接全是灰色的，这样好吗？**
A：这是预期的。灰色链接表示"知识图谱中已存在关联，但目标页面尚未展开"。只要 Lint 定期清理，这就是健康状态。

**Q：如果某个概念被引用了 10 次，但我觉得不值得独立页面怎么办？**
A：Lint 报告会列出所有缺失页面及其引用次数。你在审阅时可以直接决定跳过某些页面，或者将它们合并到其他页面中。

**Q：链接格式修复（完整路径 → 标准格式）谁来处理？**
A：Lint 阶段会检测并修复格式错误的链接。首次执行时可能涉及大量修复，后续增量很小。
